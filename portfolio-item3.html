Ansible/Ansible Tower End-to-End Automation

Designing Infrastructure as a Service

In previos projects I have already implemented complex end-to-end automation solutions with the Micro Focus Tool Chain. In this project however, Ansible/Ansible Tower is used as the base framework. With Ansible the processes for provisioning a virtual machine (VM) were fully automated, and with Ansible Tower, the automation was turned into an IT service.

With the combination of these two Red Hat products, both the communication of the surrounding systems such as phpIPAM or vCenter took place, as well as the basis configuration of the respective VM itself. Access control, order mask or even asset management were completely solved via the features of Ansible Tower.

The automation contains the following features:

  • Automatic deployment and undeployment of all assets
  • Automatic undeployment of unused VMs
  • Implemented order options like VM specs or environment
  • Support for vCenter custom templates as base system
  • Error handling with E-Mail notification
